## 설계도 & 객체지향

- Java를 배우고 설계도를 그릴 수 있으면 된다.
    - 설계도 여러 장의 관계를 어떻게 표현하는가?
    - 설계도를 그리는 것 → 여러 장을 연결하는 법, 표준을 지키는 방법 → 인터페이스

## Java 실행 구조 & 환경 변수

- 이클립스는 설치 프로그램이 아니라 메모리에 올려놓고 쓰는 것
- java는 workspace 단위로 폴더를 만듦 (or namespace)
    - 워크스페이스 바꾸면 환경은 초기화됨
- java에서 모듈 개념 거의 안 씀
- 이클립스는 `JAVA_HOME` 환경 변수 사용
    - 형식: `%환경변수%`
- 한글 깨짐 방지 → UTF-8

## 접근제어 & 캡슐화

- `modifiers`: 접근자

## annotation

- 메타데이터로 컴파일러에게 미리 요구사항을 만들어둔 것
    - getter, setter는 자동으로 생성된다
    - 컴파일러가 해줌
    - 1일 개발자와 20년 개발자 코드가 똑같다 → 해당 코드는 없어지므로 생략되어야 함
    - annotation은 멤버 필드를 못 만들어 준다
    - @Data
    - 단점: 쓸모 없는 것만 만듦
    - 성능은 상관 X, 편리성만 논하자
    - 가독성 떨어짐

→ 원리를 알자, 코드의 반복을 줄이자
