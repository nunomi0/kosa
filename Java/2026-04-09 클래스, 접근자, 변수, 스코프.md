## 클래스

### 개념

- 클래스 == 설계도 == 타입(사용자가 정의하는 큰 타입)
- ex) Emp(클래스) 타입의 emp 가능하다

### 구성요소

```java
Class 설계도 {
		고유정보
		상태정보
		부품정보 >> class (다른 설계도)
		
		+) 필요(생성자, getter, setter)
		+) 기능 행위 (method)
}
```

1. **필드 (member field)**
    - 데이터, 정보를 담고 있음
    - 고유 정보(색상), 상태 정보(속도), 부품 정보(엔진)
        - 부품은 쪼갤 수 있다
        - 별도의 클래스로 나눌 수 있다
        - ex) 자동차에서의 엔진
2. **메서드 (method)**
    - 기능(행위)를 나타냄
    - 한 번에 하나의 기능만
        - ex) 달린다, 멈춘다, 소리가 난다, 문을 연다
        - 만약 여러 개의 기능을 구현해야 한다면 다른 기능 호출 사용
    - 클래스 내부에서 사용하는 함수를 메서드라고 함
        - 자바는 클래스 내부에 함수가 있으므로 무조건 메서드
- **생성자 (constructor)**
    - 특수한 목적을 가진 함수 (기능 정해져 있다)
    - 자동화를 위함
        - 아니면 손코딩 해야한다
    - 객체 생성시 member field 초기화 → 강제성
        - ex) 아파트를 지을 때 색상과 이름은 무조건 정하고 들어갈 것이다.
        - 강제성을 주는 방법: 오버로딩
- getter, setter
    - 캡슐화된 자원에 대한 read/write
    - 직접 할당을 막고 간접 할당하는 목적

모든 함수는 호출에 의해서만 동작한다?

- X
- 생성자 함수는 자동으로 불러와짐

## 변수(variable), 스코프

static > instance > local > block

1. static
    - 가장 큰 범위 (전역)
    - 객체 간 공유 자원 (heap에 생성된 객체간 공유 자원)
    - 프로그램 종료 시 메모리에서 사라짐
    - GC(Garbage Collector)에서 수집하지 않음
2. instance
    - 클래스 내부에 있는 변수
    - 생성되는 객체마다 다른 값을 가질 수 있다
    - 객체마다 가지는 변수이기 때문에 instance 라는 이름을 붙임
    - default 가지고 있다
        - 초기화를 한다면 생성자 이용
3. local
    - 메서드 내부의 변수
    - 초기화 필요
4. block
    - 제어문 시작 시 생성, 종료 시 소멸
    - ex) for문 내의 변수

```java
class VariableClass {
	int iv;
	static int cv;
	
	void method() {
		int lv = 0;
		for (int i = 0; i<=100; i++){
		}
	}
}

```

1. int iv
    - member field, instance variable (생성되는 객체마다 다른 값을 가짐)
    - iv는 언제 메모리에 올라가는가 (사용가능한가?)
        - heap에 올라갔을 때 (new VariableClass() 했을 때
    - 목적: 데이터 자료를 담을 목적
2. static int cv
    - class Variable (클래스 변수) >> static variable
    - 객체 간 공유 자원(모든 객체가 공유하는 자원)
    - static 자원은 class 자원만 있어도 메모리에 올려버린다.
        - heap을 그리면 0점
    - `class 이름 . 자원` 형식으로 접근
    

```java
class Test { // default: 같은 패키지 안에서만 이 클래스에 접근 가능
	int i; // 인스턴스 변수 -> new Test()로 객체 만들 때 힙에 생성
	void print() {
		for (int i = 0; i<=100; i++){ // 지역 변수 -> 스택에 생성
			// print() 안에서 i를 쓰면 지역 변수가 우선
			// this.i를 쓰면 인스턴스 변수에 접근
		}
	}
}
```

- 같은 명령 공간에서 변수명이 겹치면 안된다?
    - 명령 공간 == 스코프 == 변수가 유효한 범위
    - 스코프가 다르면 같은 이름을 써도 된다
    - 메모리 공간이 다르면 같은 이름도 가능하다
        - 인스턴스 변수 → 힙에 생성
        - 지역변수 → 스택에 생성
        
- 함수의 실행은 스택이다
    - 스택의 맨 위에 있는 함수가 실행 중인 것
    - 그래서 메인함수가 마지막으로 빠지면 프로그램 종료

### static

1. static member field: 객체간 공유자원
2. static method: 객체간 공유자원
    1. 실행하려면 스택에 끌어내려야 함

메모리에 올라가는 순서

- static이 먼저 메모리에 올라감, 일반이 그 다음에 메모리에 올라감
- 최대한 static은 static끼리, 일반 자원은 일반 자원끼리 묶자
    - 일반 자원이 static에 접근하는 경우는 꽤 있다

static variable >> 공유자원 (heap 생성된 객체간 공유자원)

1. 객체 생성 이전에 memory 올라가는 자원 (클래스 or 메서드 영역)
2. 설계도 > 구체화 > new
3. static 자원은 new 없이 메모리에 올라감
    1. 메모리에 올라간 자원은 사용 가능하다.

static 접근자를 가진 클래스 변수 접근 방법

1. 클래스 이름. 자원
2. heap객체 접근 후 읽기

요구사항에 없는 코드는 작성하지말자

ex) 개별 getter 함수 작성

```java
private static int airTotalCount;
// 클래스 내부에서만 접근 가능 (클래스이름.변수이름 불가)
```

## 초기화

1. static variable: 공유자원 (객체간)
2. instance variable: 생성되는 객체마다 다른 값을 가지게 하겠다

- 두 자원은 default 값을 초기화 할 필요 없다.
- 생성자 역할은 membe field 초기화

static 자원을 초기화 시킬 수 있다?

- 초기화 시킨다는 건 메모리에 이미 자원이 올라가 있어야 함
- 생성자는 static, instance 변수 초기화가 가능하다

new를 하지 않으면 static variable 초기화가 안된다.

- static {} : static 초기자

```java
class Test {
	static int cv = 10; // 조작을 못한다
	static int cv2; // default = 0;
	
	int iv = 9;
	
	static {
		// 실행 시점: static int cv, static int cv2 클래스 영역에 올라간 직후에 실행
		cv2=cv+2222; // 조작된 값을 쓸 수 있다 (블록 {} 을 제공해야 논리가 만들어진다)
	}
	
	// 일반 자원 초기자(초기화 블록, 기능이 생성자와 겹침, 활용도가 생성자가 높음)
	{
		
	}
}
```

- 일반 초기화 블록이 실행되려면 new가 필요

1. instance variable은 초기화 하지 않아도 된다. → yes
    - default 값이 있기 때문이다
    - 초기화: 변수가 처음으로 값을 할당 받는 것
2. instance variable 초기화 해도 되나요? → yes
    - public int age = 1;
        - 필요에 따라서는 기본값(초기값)을 설정 가능
        - 문제는 없지만 근데 생성자가 할 일이 없어짐
        - 생성자가 더 풍부한 초기화를 할 수 있다
        - 목적을 어길 필요는 없다
        
        → 그러니까 생성자를 활용하자
        
3. 거의 초기화를 하지 않는다
    - 생성자를 통해 값을 받아서 초기화
    - 확장성에 대한 문제
    - 설계자의 의도를 파악해야함
        - 생성되는 객체마다 다른 값을 가졌으면 좋겠다

### 가장 전형적인 DTO 클래스

- 1일차도 만들 수 있다 → 누가 만들어도 동일한 코드 → 불필요한 코드다 → annotation으로 자동화
    - 롬복(lombok) 사용

```java
public class NoteBook {
	private int key;

	public NoteBook() {
		
	}

	public NoteBook(int key) {
		
		this.key = key;
	}

	public int getKey() {
		return key;
	}

	public void setKey(int key) {
		this.key = key;
	}

	@Override
	public String toString() {
		return "NoteBook [key=" + key + "]";
	}
	
}
```
