## Java API

- Java가 미리 만들어서 제공하는 클래스, 인터페이스 전체

```java
Java API (전체)
│
└── 컬렉션 프레임워크 (군집 데이터를 다루는 표준)
    │
    ├── Collection 인터페이스 (단일 값을 다룸)
    │   ├── List → ArrayList, Vector, LinkedList
    │   ├── Set  → HashSet, TreeSet
    │   └── Queue → LinkedList
    │
    └── Map 인터페이스 (key-value 쌍을 다룸)
        └── HashMap, Hashtable, Properties
```

## 컬렉션 프레임워크 (Collection Framework)

- 다수의 데이터를 다루는 방법을 표준화된 인터페이스로 구현하는 클래스 제공
- 컬렉션에서 가장 중요한 문법 → 다형성
    - 부모 타입의 참조 변수는 자식 객체의 주소를 받을 수 있다
    - 디커플링(유연한) 프로그래밍 만들 수 있고 유지보수에도 편리

## Collection (Interface)

- 단일 값을 다룸

#### 배열 vs 컬렉션

- 배열(Array): 다수의 군집 데이터를 다루는 방법의 가장 원시적인 방법
    - 정적 크기의 한계 → 방이 더 필요하다면 개발자가 직접 데이터 이동해야 함
        
        → ArrayList, Vector 와 같은 컬렉션 등장
        
- Java는 동적 배열을 쓰지 않음 → 정적 배열 반대는 컬렉션

| 구분 | Array (정적 배열) | Collection |
| --- | --- | --- |
| 크기 | 한 번 정해지면 변경 불가 | 동적으로 확장, 축소 |
| 확장 | 개발자가 직접 데이터 이동 | 자동 처리 |

### 1. List

제어의 논리(데이터 추가, 삭제, 조회)를 함수로 표준화한 인터페이스

#### 1-1. ArrayList

- **내부 구조**: 배열(Array) 기반
- **크기**: 동적 확장/축소 (개발자가 직접 관리할 필요 없음)
- **동기화**: 보장 안 됨 (Lock 옵션)
- **성능**: 빠름

```java
ArrayList list = new ArrayList();
list.add("A");       // 순차 추가
list.add(0, "B");    // 비순차 추가 → 데이터 이동 발생 (느림)
list.remove(0);      // 삭제 후 빈 방 없이 자동 당겨짐
list.clear();        // 데이터 삭제, 용량은 유지
Collections.sort(list); // 정렬
```

#### 1-2. Vector

- **내부 구조**: 배열(Array) 기반 (ArrayList와 동일)
- **크기**: 초기 용량 10, 다 차면 10개씩 복사 확장
    - https://docs.oracle.com/javase/8/docs/api/java/util/ArrayList.html#:~:text=public%C2%A0ArrayList()-,Constructs%20an%20empty%20list%20with%20an%20initial%20capacity%20of%20ten.,-ArrayList
- **동기화**: 기본 보장 (Lock default)
- **성능**: 느림 (Lock 오버헤드)
- **현재 거의 사용하지 않는 이유**: 단일 스레드 환경에서는 동기화가 불필요한 오버헤드이고, 멀티스레드가 필요하면 ArrayList에 Lock을 거는 방식이 더 유연하기 때문

```jsx
Vector v = new Vector();
v.capacity(); // 초기 용량: 10
v.size();     // 실제 데이터 수: 0
v.add("AA");  // 파라미터 타입: Object (오버로딩 있음)
v.add(10);    // 오토박싱: int → Integer (즉, int 10 → Integer 객체로 자동 포장(boxing)해서 Object 자리에 넣어주는 겁니다. 반대로 꺼낼 때는 언박싱(unboxing) 이 일어납니다)
```

```jsx
public boolean add(Object o)
// 파라미터 타입이 Object이기 때문에 어떤 타입이든 다 넣을 수 있습니다. String, Integer, 직접 만든 클래스 등 모든 객체가 가능합니다.ㅋ
```

#### 1-3. LinkedList

- **내부 구조**: 노드(Node) 기반 (이중 연결 리스트)
- **각 노드**: `이전 주소 | 데이터 | 다음 주소` 구조
- **동기화**: 보장 안 됨
- **적합한 경우**: 비순차적(중간) 삽입/삭제가 빈번한 경우

```jsx
LinkedList llist = new LinkedList();
llist.add("A");
llist.add(0, "B"); // 비순차 추가 → 노드 연결만 변경 (빠름)
llist.remove(0);   // 노드 연결만 변경 (빠름)\]
```

| 구분 | ArrayList | Vector | LinkedList |
| --- | --- | --- | --- |
| 내부 구조 | 배열 | 배열 | 노드(이중 연결) |
| 동기화 | X | O (기본) | X |
| 조회 속도 | 빠름 (인덱스 접근) | 빠름 | 느림 (순차 탐색) |
| 중간 삽입/삭제 | 느림 (데이터 이동) | 느림 (데이터 이동) | 빠름 (연결 변경) |
| 순차 추가/삭제 | 빠름 | 보통 | 보통 |
| 현업 사용 | **주로 사용** | 거의 안 씀 | 필요 시 사용 |

### 2. Set

- Collection 인터페이스를 상속한 인터페이스
- 순서가 없고 중복을 허락하지 않는 데이터 집합

#### 2-1. HashSet

- 단순 중복 제거 (정렬 X)
- 내부 구조: 해시 테이블 (HashMap 기반)
    - HashMap은 key 중복을 허용하지 않으므로, 이 특성을 그대로 이용해서 Set의 중복 방지를 구현. value 자리에는 의미 없는 더미 객체를 넣어둠
- 성능 빠름

```java
// 구현 클래스 타입으로 선언
HashSet<Integer> hs = new HashSet<Integer>();

// 인터페이스 타입으로 선언 -> 권장
Set<Integer> hs = new HashSet<Integer>();

while (hs.size<6){
	hs.add((int)(Math.random()*45+1));
}
```

#### 2-2. TreeSet

- 중복 제거 + 정렬
- 내부 구조: 이진 탐색 트리(Red-Block Tree)
- 정렬 비용 때문에 HashSet에 비해 상대적으로 느림

TreeSet vs HashSet 성능 비교

| 연산 | HashSet | TreeSet |
| --- | --- | --- |
| 추가 (add) | O(1) | O(log n) |
| 검색 (contains) | O(1) | O(log n) |
| 삭제 (remove) | O(1) | O(log n) |

### 3. Queue

- FIFO (First In, First Out)

| 구분 | 실패 시 예외 발생 | 실패 시 null/false 반환 |
| --- | --- | --- |
| 추가 | `add(e)` | `offer(e)` |
| 꺼내기 (제거) | `remove()` | `poll()` |
| 확인 (제거 안 함) | `element()` | `peek()` |

```java
public static void main(String[] args){
	Queue<Message> messageQueue = new LinkedList<Message>();

			break;

		...
```

## Map (Interface)

- Map은 key-value 쌍으로 데이터를 관리하는 인터페이스
- Generic 지원
    
    ```java
    HashMap<String, String> map = new HashMap<>();
    ```
    

#### Hashtable, HashMap

- Map 인터페이스를 구현하고 있는 클래스

| 구분 (클래스) | Hashtable (구버전) | HashMap (신버전) |
| --- | --- | --- |
| 동기화 | 보장 O (Lock 기본) | 보장 X (Lock 옵션) |
| 성능 | 느림 | 빠름 (현업 주력) |
- 주요 메서드

```java
HashMap map = new HashMap();
map.put("Tiger", "1004");
map.containsKey("Tiger");      // true
map.containsKey("tiger");      // false
map.get("Tiger");              // "1004"
map.get("Hong");               // null
map.size();                    // 1
map.put("Tiger", "1008");
map.toString();                // {Tiger=1008}
```

```java
// 순회 방법

// 1. key만 — keySet()
Set set = map.keySet();
Iterator it = set.iterator();
while (it.hasNext()) {
    System.out.println(it.next());
}

// 2. value만 — values()
Collection clist = map.values();

// 3. key + value 동시에 — entrySet()
for (Map.Entry m : map.entrySet()) {
    System.out.println(m.getKey() + " / " + m.getValue());
}
```

- Map 활용
    - 객체 직렬화: 객체를 key-value로 변환하여 전송/저장
    - MyBatis Parameter: SQL에 여러 값을 Map으로 묶어서 전달
        - MyBatis
            - Java에서 SQL을 편하게 다루기 위한 프레임워크
            - SQL을 XML 파일에 분리해서 작성하고 Java 코드에서는 Map으로 파라미터를 넘기기만 하면 됨
    - 채팅 서버: 접속자 ID(key)와 소켓 정보(value) 관리
    - 로그인 캐싱: DB의 ID/PWD를 메모리에 올려 빠른 인증

#### Properties

- Map 인터페이스를 구현한 클래스
- key, value 가 모두 String으로 고정된 특수한 Map

## Iterator

- 컬렉션에 저장된 데이터를 순차적으로 하나씩 꺼내는 표준화된 방법
    - 인덱스가 없는 컬렉션도 있음 → 어떤 컬렉션이든 동일한 방식으로 순회할 수 있는 표준이 Iterator

```java
List<Integer> intlist = new ArrayList<Integer>();
intlist.add(10);
intlist.add(20);
intlist.add(30);

Iterator<Integer> it2 = intlist.iterator();
while (it2.hasNext()) {
    System.out.println(it2.next());  // 10, 20, 30
}
```

| 메서드 | 반환 타입 | 설명 |
| --- | --- | --- |
| `hasNext()` | **boolean** | 다음 값이 있는지 확인 |
| `next()` | **Object** | 다음 값을 꺼내서 반환 |
| `remove()` | void | 현재 값 삭제 (추상 메서드, 구현은 각 클래스가 함) |
- 순방향만 가능
    - 역방향은 ListIterator 사용

```java
ListIterator<Integer> it = intlist.listIterator();

// 순방향
while (it.hasNext()) {
	System.out.println(it.next());
}

// 역방향
while (it.hasPrevious()) {
	System.out.println(it.previous());
}
```
