지역변수

- local variable(지역변수)은 함수 stack 내의 frame에 자리 잡음
- 지역 변수는 반드시 초기화가 선행되어야 함

## 타입

### primitive type 8가지 + String → 9가지

- byte, char, int, long, float double, boolean

### 1. 정수(음의 정수, 0, 양의 정수)

- 정수는 int
- 실수는 double
- int 보다 길다 → long
- 정수 리터럴의 기본 타입
    - 정수 리터럴: 개발자가 입력하는 정수 값
- short는 자바에서 쓰지 않음

byte란?

- 8 bit
- -128~127
- 맨 앞은 parity bit
- 자바는 빨대를 두 개 준다 (물용 vs 공차용)
    - java에서 byte를 io(입출력)에 쓴다
        - 버퍼 쓰면 io 양이 줄어든다
    - 이미지, 엑셀 파일은 바이트로 왔다갔다함
    - 한 문자를 2byte로 보자(char)
        - 영문 한 글자도 2byte, 한글, 중국어도 각각 2byte
- Ex)
    - ename varchar2(10)
        - 가변 길이는 가장 뒤에 놓는다
    - char(10)
        - 고정된 길이는 무조건 char 써야 성능이 나아짐
        - 검색이 더 빠르다
    - nvarchar2(10)
        - 한글 10자, 영어 10자, 총 20byte 크기 (n은 유니코드)

### 2. 실수(부동 소수점: 대략적인 값)

- float(4byte) 7자리 정도의 소수를 표현 가능(반올림)
- double(8byte) 15자리 정도의 소수를 표현 가능
    - 실수 리터럴의 기본 타입
        - 실수 리터럴: 개발자가 입력하는 실수 값

    - ai는 try catch를 잘 만들지 않음

char ch = ‘가’

- 문자를 저장하지만 내부적으로는 정수 값을 가지고 있음

```jsx
char ch2 = 'A';
int chint = (int)ch2;
System.out.println("ch2 문자를 정수 값으로 보면: "+ chint);
```

컴파일러는 int chint = ch2를 어떻게 해석할까?

- int chint = (int)ch2;
- (int)를 내가 명시하지 말자. 컴파일러가 싫어함

## 정리

할당에 있어서는 값을 보지 말고 값이 가지는 타입을 보자

- 리터럴 값에 대해서(개발자가 직접 입력하는 값)
    - 정수 리터럴에 기본 타입은 int
    - 실수 리터럴에 기본 타입은 double
- 형 변환 (컴파일러: 암시적, 내가 직접: 명시적)
    - 큰 타입에 작은 타입을 넣는 것은 고민하지 말자 (컴파일러가 알아서 형 변환)
    - 작은 타입에 큰 타입을 넣고자 한다면 자동 형 변환은 지원하지 않음 (강제적 형 변환)
    - 데이터 손실이 발생할 수도 있다 (해결 방법: 받는 그릇을 크게 하자)
        - ex) char c = (char)10; → 책임은 개발자가 져야함
        - ex) char c = (char)100000000; → 손실 일어남
- String 클래스 타입
    - 값 타입처럼 써도 문제 없다
    - String str = “홍길동”;
    - 자료구조 char 배열; [’홍’][’길’][’동’]
