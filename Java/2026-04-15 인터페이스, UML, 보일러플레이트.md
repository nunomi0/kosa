## 인터페이스

- 소프트웨어 설계 최상위 단계
- 약속, 규칙, 규약, 표준을 만드는 행위
- 구현부는 없음
- ~할 수 있는 (able) 로 해석하기
- 물리적 형태는 클래스와 동일 (`.java` → `.class` 컴파일)
- 생성 방법
    
    ```java
    interface Ia {
        int VERSION = 1;  // public static final 생략 가능
        void run();       // 추상함수
    }
    ```
    
    ```java
    class Test extends Object implements Ia, Ib, Ic
    ```
    

### 클래스 vs 인터페이스

| 구분 | 클래스 | 인터페이스 |
| --- | --- | --- |
| 객체 생성 | `new`로 가능 | 불가 |
| 역할 | 객체를 만드는 설계도 | 약속, 규칙, 표준을 정의 |
| 내용 | 완성된 코드 | 미완성 코드 (구현부 없음) |
| 사용 | 직접 사용 | 구현 클래스가 있어야 사용 가능 |

### 추상클래스 vs 인터페이스

- 공통 코드를 물려주려면 추상 클래스, 공통 약속만 강제하려면 인터페이스를 사용
- 인터페이스는 다중 구현이 가능하기 때문에 서로 관련 없는 클래스들에 공통 기능을 부여할 때 유리
    - 다중 구현: 하나의 클래스가 여러 인터페이스를 동시에 `implements` 하는 것
    
    ```java
    interface Flyable { void fly(); }
    interface Swimmable { void swim(); }
    
    class Duck implements Flyable, Swimmable {
        public void fly() { }
        public void swim() { }
    }
    ```
    

| 구분 | 추상클래스 | 인터페이스 |
| --- | --- | --- |
| 자원 구성 | 완성 + 미완성 | 미완성만 (상수 제외) |
| 키워드 | `extends` | `implements` |
| 다중 상속 | 불가 | 가능 |
| 변수 | 일반 변수 가능 | 상수(`public static final`)만 |
| 메서드 | 일반 + 추상 | 추상만 (JDK8부터 `default`, `static` 가능) |
- 공통점
    - 스스로 객체 생성 불가 (`new` 사용 불가)
    - 둘 다 추상 자원(실행 블럭 없음)을 갖고 있다 → 강제적 구현 목적 → 재정의
- TIP
    1. 인터페이스 나오면 무조건 다형성 입장으로 접근하자 (인터페이스는 부모 타입)
    2. 서로 연관성이 없는 클래스를 하나로 묶어주는 역할 (같은 부모를 만듦)
    3. JAVA API에 이미 수많은 인터페이스가 만들어져 있음 (활용하기)
        
        ex) `Collection` → `List,` `Set`, `Map` 모두 인터페이스이기 때문에 `new` 못 쓰고 구현 클래스 써야함
        
        ```java
        List<String> list = new ArrayList<>();   // ArrayList가 List를 구현
        Set<String> set = new HashSet<>();       // HashSet이 Set을 구현
        Map<String, Integer> map = new HashMap<>(); // HashMap이 Map을 구현
        ```
        
    4. 객체 간 연결 고리 (객체 간 소통의 역할)
        - 서로 관련 없는 클래스들이 같은 인터페이스를 구현하면, 그 인터페이스 타입 하나로 두 객체 사이의 연결 고리 역할을 함 → 다형성 원리
        - 모든 참조 타입은 `Object`의 메서드 사용 가능
            - 인터페이스 타입도 자바의 참조 타입이기 때문에 `Object`의 자원 사용 가능
            
            ```java
            class Car extends Object implements Moveable { }
            //          ^^^^^^^^^^^^^^ 모든 클래스는 Object를 상속
            ```
            

### A는 B를 사용한다

1. 상속
    
    ```java
    class A extends B { }  // A는 B를 상속받아 사용
    ```
    
2. 포함 (멤버 필드로 가짐)
    
    ```java
    class A {
        B b;  // A가 B를 멤버 필드로 가지고 있음
    }
    ```
    
    2-1. 합성(Composition)
    
    - 라이프사이클이 동일한 경우 (A가 죽으면 B도 같이 죽음)
    
    ```java
    class Car {
        Engine engine;
        Car() {
            engine = new Engine();  // Car가 생성될 때 Engine도 생성
        }                           // Car가 사라지면 Engine도 사라짐
    }
    ```
    
    2-2. 집합(Aggregation)
    
    - 라이프사이클이 다른 경우 (A가 죽으면 B는 살아있음)
    
    ```java
    class Team {
        Member member;
        void setMember(Member m) {
            this.member = m;  // 외부에서 만든 Member를 받아옴
        }                     // Team이 사라져도 Member는 존재함
    }
    ```

## UML

- 그려서 얻고자 하는 것? → 관계 파악
- 전부 그리는가? → NO / 커뮤니케이션이 필요한 경우만
- 종류 (알아야 할 것)
    - 클래스 다이어그램
        - 클래스들 간의 관계를 표현
    - 시퀀스 다이어그램
        - 객체들 간의 메시지 호출 순서를 시간 흐름에 따라 위에서 아래로 표현
    - 유스케이스 다이어그램
        - 시스템이 어떤 기능을 제공하는지, 누가 사용하는 지 표현
        - 시간의 흐름으로 가면 안 됨
- StarUML, Visual Paradigm 사용해서 작성 가능

### 접근 제어자 표기

```
+ public

```

### 관계 판단 기준

- 멤버 필드 선언 X → 의존(dependency)
- 멤버 필드 선언O → 연관(association)
    - 생성자에서 초기화 → 합성(compotision)
    - 일반 함수에서 초기화 → 집합(aggregation)

## String

- 불변객체: 한 번 만든 메모리는 못 고친다
- 누적하지 않는다
- String은 Char 배열로 데이터를 관리 → `charAt` 사용 가능
- StringBuffer vs StringBuilder
    - 문자열을 누적(수정)할 수 있는 객체
    
    | 구분 | StringBuffer | StringBuilder |
    | --- | --- | --- |
    | 속도 | 느림 | 빠름 |
    | 동기화 | 있음 (lock 가짐) | 없음 |
    | 안전성 | 멀티스레드 환경에서 안전 | 싱글스레드에서 사용 |

## 일급 객체

- 함수를 값처럼 취급할 수 있는 것
    1. 함수를 변수에 할당할 수 있다
    2. 함수를 다른 함수의 인자로 전달할 수 있다
    3. 함수를 다른 함수의 리턴값으로 반환할 수 있다
- JavaScript가 대표적

## 보일러플레이트

- 의미있는 로직이 아님, 반복적으로 항상 작성해야 하는 코드
- 보일러플레이트를 줄이기 위해 Lombok, Spring, Record 사용
    - Lombok: 어노테이션 하나로 getter, setter, 생성자 등을 자동 생성해주는 라이브러리
    - Spring: 프레임워크 레벨, 객체 생성, 의존성 주입, 설정 코드 등을 어노테이션으로 자동 처리
    - Record: 불변 데이터 객체를 한 줄로 만들 수 있는 자바 기능 (JDK16+)
